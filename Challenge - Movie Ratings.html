#Ploting ROC curve
#Input: object of the algorithm, Test set.
#Output: ROC Curve.
def plot_roc(algorithm, X_test, y_test):
    y_score = algorithm.predict(X_test)
    fpr, tpr, thresholds = roc_curve(y_test, y_score)
    auc = roc_auc_score(y_test, y_score)

    plt.plot(fpr, tpr, label="ROC Curve (area = %0.2f)" % auc, color='r')
    plt.plot([0, 1], [0, 1], color='navy', lw=2, linestyle='--')
    plt.xlim([0.0, 1.0])
    plt.ylim([0.0, 1.05])
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('Receiver Operating Characteristic')
    plt.legend(loc="lower right")
    plt.savefig('ROC GBM noshows.png')
    plt.show()

    print('Area under ROC: '+str(auc))

#Ploting feature importance
#Input: object of the algorithm, Test set.
#Output: Feature importance graph.
def plot_feature_importance(alg, X_test, fig_size=None):
    n_features = X_test.shape[1]
    if fig_size == None:
        fig = plt.figure(figsize=(int(n_features),4))
    else:
        fig = plt.figure(figsize=fig_size)
    feat_imp = pd.Series(alg.feature_importances_, X_test.columns).sort_values(ascending=False)
    feat_imp.plot(kind='bar', title='Feature Importances')
    plt.ylabel('Feature Importance Score')
    
    plt.show()

Siguientes pasos¶

Clculo correcto de promedios de usuarios y pelculas. Tomando en cuenta slo aquellos datos antes de la observacin en cuestin.

Normalizacin y/o logaritmo de las variables para reducir variabilidad.

Incluir informacin del idioma de la pelcula y presupuesto gastado, no es una regla pero se piensa que aquellas pelculas con un budget grande tienden a tener buenos ratings.

Intentos fallidos¶

Cluster de gneros de pelculas. Se intent hacer una clusterizacin basada en los genros pero la mtrica silhouette no fue satisfactoria por lo que se sigi con otro camino.

Conclusiones¶

Fue primordial haber incluido informacin acerca del staff de la pelcula. Sin estos features el clasificador no tena un buen desempeo.

Se invirti una mayor cantidad de tiempo en hacer feature engineering pues es lo ms importante ante un problema prediccin ya que se debe alimentar el modelo con datos muy significativos y explicativos que ayuden a que el algoritmo usado pueda detectar patrones de mejor forma.

En un inicio se prob con una regresin logstica pero no se obtuvo un buen desempeo, se decidi probar con un modelo ms robusto (Gradient Boosting) pues no es un problema bsico.
